# 바이브코딩 아이디어 관리 - PRD (Product Requirements Document)

## 프로젝트 개요

**프로젝트명**: 바이브코딩 아이디어 관리  
**목적**: Notion을 CMS로 활용하여 생각날 때 상세한 바이브 코딩 아이디어를 체계적으로 관리하는 웹 플랫폼

**CMS 선택 이유**:
- Notion API를 활용하여 비개발자도 콘텐츠 관리 가능
- 직관적인 UI로 빠른 정보 입력 가능
- 별도의 백엔드 데이터베이스 관리 불필요
- 아이디어 기재 시각 자동 기록

---

## 주요 기능

### 1. 아이디어 목록 관리
- Notion 데이터베이스에 기재된 아이디어를 웹에서 조회
- 아이디어를 **No 순서로 정렬**하여 일관된 관리
- 각 아이디어마다 기재한 시각을 함께 표시
- 카테고리별, 상태별 필터링 기능 (MVP 후 추가 검토)

### 2. 아이디어 상세 조회
- 아이디어 제목, 상세 내용, 카테고리, 상태 확인
- 생성 시각 및 마지막 수정 시각 표시

## Notion 데이터베이스 구조

### 필드 정의

| 필드명 | 필드 타입 | 설명 | 필수 여부 |
|---|---|---|---|
| **No** | Number | 아이디어 고유 번호 (목록 정렬 기준, 수동 입력) | ✅ 필수 |
| **Title** | Title | 아이디어 제목 | ✅ 필수 |
| **Description** | Rich Text | 아이디어 상세 내용 | ✅ 필수 |
| **Category** | Select | 카테고리 (프론트엔드, 백엔드, 풀스택, 모바일, 기타) | ✅ 필수 |
| **CreatedAt** | Date | 기재 시각 (Notion 자동 생성 타임스탬프 또는 수동 입력) | ✅ 필수 |
| **Status** | Select | 상태 (아이디어, 진행중, 완료, 보류) | ✅ 필수 |

### 데이터베이스 필터/정렬 규칙
- **정렬**: No (오름차순) - 가장 오래된 아이디어부터 최신 아이디어 순으로 표시
- **필터** (향후 추가): Status가 "보류"가 아닌 항목만 표시 (선택 사항)

## 화면 구성

### 1. 홈 페이지 (아이디어 목록)
**URL**: `/`

**레이아웃**:
- **Header**: 프로젝트 제목, 테마 토글
- **Main Content**: 
  - 아이디어 카드 그리드 (No 오름차순 정렬)
  - 각 카드 정보:
    - 제목 (Title)
    - 요약 (Description 처음 100자)
    - 카테고리 배지
    - 상태 배지
    - 기재 시각 (CreatedAt)
  - 카드 클릭 시 상세 페이지로 이동
- **Footer**: 프로젝트 정보

**기술 구현**:
- Notion API로 데이터 베이스 조회 (서버 사이드)
- shadcn/ui의 Card, Badge 컴포넌트 활용
- Lucide React 아이콘으로 카테고리별 구분 시각화

### 2. 아이디어 상세 페이지
**URL**: `/ideas/[id]`

**레이아웃**:
- **Header**: 뒤로가기 버튼, 아이디어 제목
- **Main Content**:
  - 제목 (Title)
  - 카테고리 배지
  - 상태 배지
  - 기재 시각 및 수정 시각
  - 상세 내용 (Description, Rich Text 렌더링)
- **Footer**: 목록으로 돌아가기 버튼

**기술 구현**:
- Notion API로 단일 페이지 조회
- Rich Text 렌더링 (마크다운 또는 HTML로 변환)

## 구현 단계

### Phase 1: 환경 설정
1. Notion API 키 발급 (Notion Developer Console에서)
2. 목표 데이터베이스 ID 확인
3. `.env.local` 파일에 API 키 저장
   ```
   NEXT_PUBLIC_NOTION_DATABASE_ID=<database-id>
   NOTION_API_KEY=<api-key>
   ```

### Phase 2: Notion 클라이언트 유틸리티 구현
1. 파일 생성: `lib/notion.ts`
2. Notion 클라이언트 초기화
3. 함수 구현:
   - `getAllIdeas()` - 모든 아이디어 조회 (No 오름차순)
   - `getIdeaById(id)` - 단일 아이디어 조회
   - TypeScript 타입 정의

### Phase 3: 아이디어 목록 페이지 구현
1. `app/page.tsx` 리팩토링
2. 서버 컴포넌트로 Notion API 호출
3. `components/IdeaCard.tsx` 컴포넌트 생성
4. 그리드 레이아웃 적용

### Phase 4: 아이디어 상세 페이지 구현
1. `app/ideas/[id]/page.tsx` 생성
2. 동적 라우팅 설정 (`generateStaticParams`, `generateMetadata`)
3. Rich Text 렌더링 로직
4. 뒤로가기 네비게이션

## 데이터 흐름

```
사용자 접속
    ↓
GET / (Next.js 페이지)
    ↓
lib/notion.ts - getAllIdeas() 호출
    ↓
Notion API - 데이터베이스 쿼리 (No ASC)
    ↓
결과를 컴포넌트로 렌더링
    ↓
IdeaCard 컴포넌트 그리드 표시
    ↓
카드 클릭
    ↓
GET /ideas/[id]
    ↓
lib/notion.ts - getIdeaById(id) 호출
    ↓
Notion API - 페이지 조회
    ↓
상세 내용 렌더링
```

## 검증 및 테스트 계획

### 개발 환경 테스트
1. `npm run dev` 실행
2. 홈페이지 (`/`)에서 아이디어 목록 렌더링 확인
3. 모든 카드가 No 오름차순으로 정렬되었는지 확인
4. 기재 시각이 올바르게 표시되는지 확인

### 상세 페이지 테스트
1. 목록의 카드 클릭
2. `/ideas/[id]` 페이지로 이동 확인
3. 상세 내용이 정상 렌더링되는지 확인
4. Rich Text 형식이 제대로 표시되는지 확인

### Notion 연동 테스트
1. Notion 데이터베이스에 새 아이디어 추가
2. 웹에서 새로고침 후 목록에 반영되는지 확인
3. 아이디어 수정 후 변경사항이 웹에 반영되는지 확인

### 성능 테스트
- Notion API 응답 시간 모니터링
- ISR(Incremental Static Regeneration) 설정으로 캐싱 성능 최적화
