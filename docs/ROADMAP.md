## Phase 1: 프로젝트 골격 (구조, 환경 설정)

**목표**: 개발 환경 완성 및 기본 구조 구축

### 📌 할 작업
- [ ] Notion API 키 발급 및 설정
- [ ] `.env.local` 파일 생성 및 환경변수 입력
- [ ] Notion 데이터베이스 생성 (필드: No, Title, Description, Category, CreatedAt, Status)
- [ ] 프로젝트 폴더 구조 확인 (app/, components/, lib/, types/ 등)
- [ ] package.json 의존성 확인 (@notionhq/client 설치 여부 확인)

### 🤔 왜 이 순서인가?

**핵심 이유**: 모든 개발의 기초 토대

1. **의존성 문제**: Notion API 키 없이 어떤 기능도 동작할 수 없음
2. **폴더 구조**: 파일 조직이 없으면 코드 관리 어려움
3. **환경 설정**: 이후 모든 단계가 이 설정을 기반으로 함
4. **병렬화 불가**: 환경이 없으면 다른 작업을 진행할 수 없음

따라서 가장 먼저, 그리고 가장 철저하게 검증해야 합니다.

### ⏱️ 예상 소요 시간
- API 키 발급: 10분
- 데이터베이스 생성: 15분
- 테스트 데이터 입력: 15분
- **소계: 약 40분**

### ✅ 완료 기준
- [x] `.env.local` 파일이 정상 로드됨
- [x] Notion 데이터베이스가 생성됨 (테스트 데이터 5개 이상)
- [x] API 키로 Notion API 호출 성공
- [x] 최신 커밋 푸시 완료

**문제 해결**:
- API 키 401 에러 → Integration 권한 재확인
- 데이터베이스 404 에러 → 데이터베이스 ID 복사본 확인

- [ ] `constants/statuses.ts` - 상태 상수 정의

### 🤔 왜 이 순서인가?

**핵심 이유**: 모든 기능의 기반이 되는 도구 준비

1. **재사용성**: 공통 모듈을 먼저 만들면 모든 기능에서 활용 가능
2. **개발 속도**: 타입 정의, 유틸리티가 준비되면 기능 개발이 빨라짐
3. **중복 제거**: 여러 곳에서 쓰는 로직을 미리 만들어 코드 중복 방지
4. **유지보수**: 공통 부분을 한 곳에서만 관리하면 버그 수정이 쉬움
5. **의존도 관리**: 핵심 기능들이 이 공통 모듈에만 의존하도록 설계

예를 들어, `lib/notion.ts`가 없으면 목록 페이지와 상세 페이지에서 각각 API 호출 로직을 중복 작성해야 합니다.

- UI 컴포넌트: 1시간
- 타입/유틸리티: 30분
- **소계: 약 2.5시간**

#### 5-3. CI/CD & 배포 (높음)
- [ ] GitHub Actions 워크플로우 설정
  - 푸시 시 자동 테스트 & 린트
  - PR 생성 시 자동 빌드 체크
- [ ] Vercel 배포 설정
- [ ] 커스텀 도메인 설정 (선택)
- [ ] 배포 자동화 및 롤백 전략

#### 5-4. 모니터링 & 분석 (낮음)
- [ ] 에러 추적 (Sentry, Error Boundary)
- [ ] 사용자 행동 분석
- [ ] 성능 모니터링 (Real User Monitoring)

### 🤔 왜 이 순서인가?

**핵심 이유**: 안정적인 프로덕션 환경 구축

1. **기능 안정성 먼저**: 기능이 100% 완성되기 전 최적화는 낭비
2. **성능은 측정 후**: 추측하지 말고 실제 데이터 기반 최적화
3. **배포 자동화**: 수동 배포는 오류 가능성 높음 → 자동화로 안정성 확보
4. **프로덕션 안정성**: 배포 후 모니터링으로 실시간 이슈 대응
5. **지속 가능성**: 자동화 인프라가 없으면 유지보수 어려움

예: 성능 최적화를 너무 일찍 하면 불필요한 복잡도 증가 (과도한 최적화는 악)

### ⏱️ 예상 소요 시간
- 성능 최적화: 2시간
- 보안 & 안정성: 1시간
- CI/CD & 배포: 1.5시간
- 모니터링: 0.5시간
- **소계: 약 5시간**

### ✅ 완료 기준
- [ ] Lighthouse 성능 점수 90 이상
- [ ] 필터링 & 검색이 정상 동작
- [ ] 키보드만으로 전체 사이트 네비게이션 가능
- [ ] Vercel에 배포되고 공개 URL 접근 가능
